Go 后端的话,情况通常更简单——多半是你的程序之前是手动跑的(或者用 `nohup`、`screen` 之类临时挂着),重启后进程没了,前端自然连不上。

按这个排查:

**1. 先确认进程在不在**
```bash
ps aux | grep 你的程序名
```
或者直接查端口:
```bash
ss -tlnp | grep 8080    # 换成你实际监听的端口
```
如果什么都没有,就是进程没起来,这是最常见的情况。

**2. 手动启一下看能不能跑起来**

先别急着后台运行,直接在前台跑,这样能立刻看到报错:
```bash
cd /你的项目目录
./你的二进制文件
```
重点看启动日志会不会报错。Go 后端重启后起不来,**最常见的真凶是依赖服务没就绪**:
- 连不上数据库(MySQL/Postgres 还没启动好,你的程序就尝试连接然后退出)
- 连不上 Redis
- 端口被占用(`address already in use`)

如果是依赖没起,先把数据库、Redis 这些拉起来,再启 Go 程序。

**3. 配成 systemd 服务(强烈建议)**

如果你之前是 `nohup ./app &` 或者 `screen` 挂着的,这就是每次重启都断的根本原因。用 systemd 一劳永逸:

新建 `/etc/systemd/system/myapp.service`:
```ini
[Unit]
Description=My Go Backend
After=network.target mysql.service    # 如果依赖数据库,加上它

[Service]
Type=simple
WorkingDirectory=/你的项目目录
ExecStart=/你的项目目录/你的二进制文件
Restart=always
RestartSec=5
User=你的用户名
# 如果用环境变量配置,在这里加:
# Environment=DB_HOST=localhost
# 或者 EnvironmentFile=/你的项目目录/.env

[Install]
WantedBy=multi-user.target
```

然后:
```bash
systemctl daemon-reload
systemctl enable myapp      # 设开机自启
systemctl start myapp       # 立即启动
systemctl status myapp      # 看状态
journalctl -u myapp -f      # 实时看日志
```

`After=mysql.service` 和 `Restart=always` 这两行很关键:前者保证数据库先起来,后者保证程序崩了会自动重启。配好之后下次服务器重启就不用管了。

## 这次到底发生了什么

一句话:**欠费停机 → 重启 → MongoDB 因为一个配置笔误起不来 → 后端连不上它就崩 → app 全线 502。**

但要注意,这里其实是**两层问题叠加**,得分开看,因为避免的方法不一样:

**第一层(导火索):欠费停机。** 这是外部触发,本身不致命 —— 服务器重启是常事。真正的问题是重启**暴露**了下面这个一直潜伏的隐患。

**第二层(真正的病根):MongoDB 的 systemd 配置有个路径笔误。** 服务文件里让 systemd 去 `/www/server/mongodb/mongod.pid` 找 PID 文件,但 MongoDB 实际把 PID 写到了 `/www/server/mongodb/log/configsvr.pid`。两个路径对不上,systemd 永远等不到,就判定"启动超时",把已经跑起来的 MongoDB 反复 kill —— 无限循环,27017 端口时有时无。

## 编译是什么

你的充值管理源码,现在是**人能读、但浏览器不能直接用**的状态。

打个比方:**源码就像菜谱,编译就是按菜谱把菜做出来,浏览器要的是那盘菜,不是菜谱。**

- 你手上那套 `admin` 源码(`.vue`、`.ts` 文件)= **菜谱**,写着"充值管理这个页面长什么样、怎么工作"
- 浏览器看不懂菜谱,它只认做好的**成品**(压缩过的 `.js`、`.css` 文件,就是你线上 `assets/` 目录里那一堆 `index-哈希.js`)
- **编译** = 把菜谱(源码)做成成品(浏览器能跑的文件)这个加工过程
- 干这个加工活的工具,就叫**编译器**(在你这个项目里,具体是 `vite` 这个工具,通过 `pnpm build` 命令来跑)

所以 `pnpm build` 这条命令的意思就是:"把我修好的源码,加工成浏览器能用的成品,放进 `dist/` 文件夹"。然后你把这个 `dist/` 传到服务器,充值管理就回来了。
