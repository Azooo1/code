这次的情况和上一条不一样了——**连 TCP 连接都没建立起来就被拒绝了。**

`connection refused`（连接被拒绝）的意思是：你的请求到达了那台机器，但 22 端口上**没有程序在监听**，所以对方直接回绝了连接。这和上次的"密码错误"是完全不同阶段的问题——上次至少握手成功了，这次连门都没敲开。

**最常见的几个原因：**

1. **SSH 服务（sshd）没在运行或挂了** — 这是最典型的。可能服务崩了、或者服务器重启后 sshd 没自动起来。

2. **SSH 改了端口** — 服务可能从 22 改到了其他端口（比如 2222），所以 22 端口没人应答。

3. **服务器正在重启 / 还没启动完成** — 如果你刚重启过这台机器，sshd 可能还没起来，等一两分钟可能就好了。

4. **防火墙规则变化** — 不过要注意：通常防火墙拦截（比如安全组没放行）表现为**超时 timeout**而不是 refused。Refused 更多指向"端口确实没在监听"。

**怎么排查：**

- **先等等再试** — 如果刚重启过，等 1-2 分钟再连
- **去云服务商控制台** — 这台是 `43.99.105.115`，看 IP 段像是腾讯云。可以登录控制台用 **VNC / 网页终端**直接进系统（不走 SSH），然后检查：
  ```

  systemctl start sshd       # 如果没运行就启动它
  ss -tlnp | grep :22        # 看 22 端口是否在监听
  ```
- **确认端口对不对** — 检查一下 SSH 配置里的端口设置（`/etc/ssh/sshd_config` 里的 `Port`）
